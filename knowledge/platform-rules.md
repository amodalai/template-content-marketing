---
title: Platform-Specific Rules
---

## Twitter/X

- Max 280 characters per tweet
- Threads: first tweet is the hook, link in a reply not in the thread
- No hashtags (dev Twitter norm)
- Images/screenshots get more engagement than text-only
- Best posting times: 8-10am EST weekdays
- External links are suppressed in ranking — ask a question instead
- Quote tweets with commentary outperform retweets

## Reddit

- Self-posts (text) get more engagement than link posts on technical subs
- Title: descriptive and direct, no clickbait
- Body: open with the problem, explain the approach, end with a question
- Never use marketing language: "platform" → "tool", "solution" → "approach"
- Comment in the thread after posting — Reddit rewards OP engagement
- Account needs organic comment history before posting
- Don't post and delete — mods track this
- Each subreddit has its own rules — always check sidebar

## Hacker News

- Title must match the article title (mods enforce this)
- Show HN posts must have a working demo
- Comments must be substantive — one-liners get flagged
- Self-promotion is OK if the content is genuinely interesting
- Never ask for upvotes
- Best time to post: 8-9am EST weekdays
- If it doesn't hit front page in 30 minutes, it probably won't

## Dev.to

- Markdown format, supports code blocks and images
- Use canonical_url pointing back to your blog (preserves SEO)
- 4 tag limit per article
- Cross-post 2-3 days after original publication
- Dev.to community is supportive — less hostile than Reddit/HN

## LinkedIn

- Longer posts perform well (500-1000 words)
- First line must hook — only 2 lines visible before "see more"
- No links in the post body (suppressed) — put link in first comment
- Professional tone, but conversational
- Good for playbooks and strategy content (not quickstarts)
