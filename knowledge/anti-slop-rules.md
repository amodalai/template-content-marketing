# Anti-Slop Writing Rules

## Overview
Explicit guidelines to prevent AI-generated content patterns that sound robotic, generic, or overly enthusiastic. These rules apply to all content creation. Do not use em-dashes.

## Banned Phrases - Never Use These

### Hype/Excitement Endings
- "just got serious" / "things just got serious"
- "and it's incredible" / "and it's amazing"
- "this changes everything"
- "mind = blown" / "mind blown"
- "plot twist:" / "here's the kicker:"
- "buckle up" / "hold onto your hats"
- "you won't believe what happened next"

### Generic AI Transitions  
- "here's the thing"
- "at the end of the day"
- "the bottom line is"
- "let me break it down for you"
- "let's dive in" / "let's dig in"
- "without further ado"
- "spoiler alert:"

### Weak Comparison Openings
- "Most [category] do X, but..."
- "While other [tools/frameworks/solutions] focus on..."
- "Unlike traditional approaches..."
- "Where other [systems] fall short..."
- "The problem with existing [solutions] is..."
- "Other [frameworks] try to..."

### Overused Intensifiers
- "absolutely incredible/amazing/revolutionary"
- "literally game-changing"
- "totally transformed"
- "completely revolutionized"
- "definitely a game changer"
- "hands down the best"

### Cliché Tech Speak
- "takes it to the next level"
- "levels up your workflow" 
- "supercharge your productivity"
- "turbocharge your development"
- "streamline like never before"
- "optimize to the max"

## Writing Style Requirements

### Be Specific, Not Dramatic
- ❌ "This architecture just got serious"
- ✅ "This architecture handles 10K concurrent connections"

### Lead with Value, Not Comparisons
- ❌ "Most agent frameworks struggle with context management, but..."
- ✅ "Context isolation prevents agent conversations from bleeding into each other"
- ❌ "While other tools focus on orchestration..."
- ✅ "Configuration-driven agents scale without code changes"

### Use Facts, Not Feelings
- ❌ "You'll love how clean this code is"
- ✅ "This reduces the codebase by 40%"

### Show, Don't Tell
- ❌ "This amazing new approach changes everything"
- ✅ "Instead of 50 lines of config, you write 3"

### End with Value, Not Hype
- ❌ "And that's when things got interesting..."
- ✅ "This saves 2 hours per deployment"

## Acceptable Alternatives

Instead of hype words, use:
- **Concrete metrics**: "reduces latency by 200ms"
- **Specific benefits**: "eliminates manual configuration"  
- **Clear comparisons**: "faster than the previous approach"
- **Factual observations**: "this pattern handles edge cases"
- **Direct questions**: "what would you build with this?"

## Test Your Content

Before publishing, ask:
1. Could this have been written by a human in 2019? (Before AI content became common)
2. Does it contain specific, falsifiable claims?
3. Would a skeptical developer find this credible?
4. Does it sound like something you'd actually say in conversation?

If any answer is "no", rewrite it.

## When to Use This
Reference these rules for all content creation, especially:
- Twitter/X threads and posts
- Article conclusions
- Product announcements  
- Technical explanations
- Any public-facing content
